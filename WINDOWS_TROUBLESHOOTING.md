# Решение проблем на Windows

## Требования к системе

### Минимальные требования:
- **ОЗУ (RAM)**: 4 ГБ
- **Процессор**: Двухъядерный
- **Место на диске**: 2 ГБ
- **ОС**: Windows 10 или новее

### Рекомендуемые требования:
- **ОЗУ (RAM)**: 8 ГБ или больше
- **Процессор**: Четырехъядерный
- **Место на диске**: 5 ГБ
- **ОС**: Windows 11

### Оптимальные требования (для локальных AI моделей):
- **ОЗУ (RAM)**: 16 ГБ или больше
- **Процессор**: Многоядерный (6+ ядер)
- **Место на диске**: 10 ГБ

## Проблема: GUI не запускается / окно не видно

### Причина 1: Нехватка памяти (маловероятно, но возможно)

**Симптомы:**
- Приложение запускается (видно в диспетчере задач)
- Но окно не появляется
- В логах нет ошибок

**Решение:**
1. Откройте **Диспетчер задач** (Ctrl+Shift+Esc)
2. Проверьте использование памяти
3. Если используется >90% памяти, закройте другие приложения
4. Перезапустите приложение

**Проверка памяти:**
```cmd
# В PowerShell
Get-CimInstance Win32_OperatingSystem | Select-Object TotalVisibleMemorySize, FreePhysicalMemory
```

### Причина 2: Окно создано, но не показывается (наиболее вероятно)

**Симптомы:**
- В логах видно: "Window is now visible"
- Но окно не видно на экране
- Процесс работает в диспетчере задач

**Решения:**

#### 1. Проверьте, не скрыто ли окно
- Нажмите **Alt+Tab** - возможно окно за другими приложениями
- Проверьте все виртуальные рабочие столы (Win+Tab)

#### 2. Проверьте, не за пределами экрана
- Нажмите **Win+Shift+Стрелка влево/вправо** - переместит окно между мониторами
- Или используйте **Win+Стрелки** для перемещения окна

#### 3. Принудительно покажите окно через DevTools
Создайте файл `.env` рядом с `.exe` и добавьте:
```env
AUTO_OPEN_DEVTOOLS=true
```
Это откроет DevTools, что может помочь окну появиться.

#### 4. Попробуйте запустить с флагом отладки
```cmd
# В командной строке в папке с .exe
electron.exe --enable-logging --log-level=0
```

### Причина 3: Проблемы с прозрачным окном на Windows

**Симптомы:**
- Окно создается, но полностью прозрачное
- Видно только в диспетчере задач

**Решения:**

#### 1. Отключите прозрачность окон в Windows
1. Откройте **Параметры Windows** → **Персонализация** → **Цвета**
2. Отключите **Эффекты прозрачности**

#### 2. Обновите драйверы видеокарты
- NVIDIA: [nvidia.com/drivers](https://www.nvidia.com/drivers)
- AMD: [amd.com/support](https://www.amd.com/support)
- Intel: [intel.com/content/www/us/en/download-center/home.html](https://www.intel.com/content/www/us/en/download-center/home.html)

### Причина 4: Антивирус блокирует отображение

**Симптомы:**
- Приложение запускается
- Но окно не появляется
- В антивирусе есть предупреждения

**Решение:**
1. Добавьте приложение в исключения антивируса
2. Или временно отключите антивирус для проверки

## Диагностика

### Шаг 1: Проверьте, запущено ли приложение
```cmd
# В PowerShell
Get-Process | Where-Object {$_.ProcessName -like "*electron*" -or $_.ProcessName -like "*meeting-notes*"}
```

### Шаг 2: Проверьте логи
Логи обычно находятся в:
- `%APPDATA%\Meeting Notes Coder\logs\`
- Или в папке с `.exe` файлом

### Шаг 3: Запустите с подробным логированием
Создайте файл `.env` рядом с `.exe`:
```env
AUTO_OPEN_DEVTOOLS=true
NODE_ENV=development
```

### Шаг 4: Проверьте использование ресурсов
```cmd
# В PowerShell
Get-Process | Where-Object {$_.ProcessName -like "*electron*"} | Select-Object ProcessName, @{Name="Memory(MB)";Expression={[math]::Round($_.WS/1MB,2)}}
```

## Быстрое решение

Если ничего не помогает:

1. **Удалите сохраненное состояние окна:**
   - Удалите файл в `%APPDATA%\Meeting Notes Coder\window-state.json`
   - Или переименуйте папку `%APPDATA%\Meeting Notes Coder`

2. **Запустите от имени администратора:**
   - Правой кнопкой на `.exe` → **Запуск от имени администратора**

3. **Проверьте совместимость:**
   - Правой кнопкой на `.exe` → **Свойства** → **Совместимость**
   - Попробуйте "Запустить программу в режиме совместимости" для Windows 10

## Контакты

Если проблема не решается, соберите следующую информацию:
1. Версия Windows (Win+R → `winver`)
2. Количество RAM (Диспетчер задач → Производительность)
3. Логи из `%APPDATA%\Meeting Notes Coder\logs\`
4. Скриншот диспетчера задач с процессами приложения
